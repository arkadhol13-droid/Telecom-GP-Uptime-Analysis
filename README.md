# Telecom GP Uptime Analysis 📊

This project analyzes Telecom GP (Gigabit Passive) uptime data using Python.  
The goal is to understand network uptime performance and visualize trends across months.

## 📌 Project Objective
To analyze monthly GP uptime distribution and identify trends in:

- High Uptime (≥98%)
- Medium Uptime (90–98%)
- Low Uptime (<90%)

This helps in identifying network performance and areas where uptime improvement is required.

---

## 🛠 Tools & Technologies

- Python
- Pandas (Data Cleaning)
- Matplotlib (Trend Visualization)
- Plotly (Interactive Dashboard)
- Jupyter Notebook

---

## 📂 Dataset

The dataset contains monthly GP uptime counts categorized into three groups:

- High Uptime
- Medium Uptime
- Low Uptime

Months analyzed:

April → February

---

## 🔎 Data Processing Steps

1. Import dataset using Pandas
2. Remove unwanted columns
3. Clean column names
4. Convert values to numeric format
5. Extract uptime categories
6. Analyze trends month-wise

---

## 📈 Visualizations Created

### 1️⃣ High Uptime Trend
Line chart showing GP count with uptime ≥98%.

### 2️⃣ Medium Uptime Trend
Line chart showing GP count with uptime between 90–98%.

### 3️⃣ Low Uptime Trend
Line chart showing GP count with uptime <90%.

### 4️⃣ Monthly Uptime Distribution
Stacked bar chart showing the distribution of all uptime categories per month.

---

## 📊 Key Insights

- High uptime GPs show network stability trends.
- Medium uptime indicates moderate performance.
- Low uptime helps identify problematic network areas.
